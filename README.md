# Game Motion + HP Detector

基于传统计算机视觉的游戏画面怪物检测原型。通过**运动检测 + 血条检测**双重条件，识别游戏中的怪物候选目标。

当前版本仅做**检测 + 可视化验证**，不包含任何鼠标键盘控制、内存读取或注入操作。

## 环境要求

- Python 3.10+
- Windows 10/11
- 屏幕分辨率建议 1920x1080（可在 config.py 调整）

## 安装

```bash
cd game_motion_hp_detector
pip install -r requirements.txt
```

## 准备 hp_template.png

将你要检测的血条截图放置到 `assets/hp_template.png`。

**制作方法：**
1. 运行目标游戏，找到一个有血条的怪物
2. 使用截图工具（如 Windows 截图、Snipaste）截取血条部分
3. 只保留血条本体，尽量去掉背景
4. 建议尺寸：宽 60-150px，高 5-15px
5. 保存为 PNG 格式到 `assets/hp_template.png`

> 如果没有模板图片，程序仍可运行，会自动跳过模板匹配，仅使用颜色检测。

## 修改截图区域

编辑 `config.py`：

```python
# 全屏模式
CAPTURE_MODE = "fullscreen"

# 指定区域模式
CAPTURE_MODE = "region"
CAPTURE_REGION = (100, 100, 800, 600)  # (left, top, width, height)

# 指定窗口模式（需安装 pygetwindow）
CAPTURE_MODE = "window"
CAPTURE_WINDOW_TITLE = "游戏窗口标题"
```

## 运行

```bash
python main.py
```

**快捷键：**
- `q` - 退出
- `p` - 暂停/继续
- `s` - 保存当前调试截图到 `output/`

## 界面说明

- **蓝框** - 运动检测区域
- **红框** - 检测到的血条区域
- **绿框** - 最终判定为怪物的目标
- **青色细线框** - 血条搜索区域
- 左上角显示 FPS 和统计信息
- 每个目标旁边显示模板匹配分数和颜色检测结果

## 调参建议

### 运动检测相关
| 参数 | 说明 | 建议 |
|------|------|------|
| `MOTION_THRESHOLD` | 帧差二值化阈值 | 画面变化大时调高（30-50），静态场景调低（15-25） |
| `MOTION_MIN_AREA` | 最小运动面积 | 远景小怪调低（200），近景大怪调高（1000+） |
| `MOTION_BLUR_KSIZE` | 模糊核大小 | 噪点多时调大（7-9），保留细节时用 3-5 |
| `MOTION_DILATE_ITERATIONS` | 膨胀次数 | 目标碎片化时调大，过度合并时调小 |

### 血条检测相关
| 参数 | 说明 | 建议 |
|------|------|------|
| `HP_SEARCH_OFFSET_TOP` | 向上搜索偏移 | 根据血条距离怪物头顶的像素距离调整 |
| `HP_SEARCH_HEIGHT` | 搜索区域高度 | 血条较高时增大 |
| `HP_TEMPLATE_THRESHOLD` | 模板匹配阈值 | 误报多调高（0.7-0.8），漏报多调低（0.4-0.5） |
| `HP_COLOR_MODE` | 颜色模式 | 红色血条用 "red"，绿色用 "green"，混合用 "both" |
| `HP_MIN_WIDTH` | 血条最小宽度 | 根据游戏中血条实际宽度调整 |
| `HP_DECISION_MODE` | 判定策略 | 调试阶段建议 "or"，精度要求高时用 "and" |

## 当前局限性

1. **帧差法的天然限制** - 静止的怪物无法被检测到
2. **背景运动干扰** - 场景中的粒子特效、UI 动画可能产生误报
3. **固定颜色阈值** - 不同游戏的血条颜色可能不在预设范围内
4. **单尺度模板** - 模板匹配不支持不同大小的血条
5. **无目标追踪** - 每帧独立检测，无 ID 关联
6. **性能限制** - 全屏截图在高分辨率下 FPS 较低

## 后续扩展方向

- **自动点击**：对确认的目标发送鼠标点击事件
- **多目标追踪**：使用 SORT / DeepSORT 对检测结果做跨帧关联
- **YOLO 二次识别**：用 YOLOv8 替代或辅助传统 CV 检测
- **血条模板多尺度匹配**：支持不同距离/大小的血条
- **背景建模**：用 MOG2 替代简单帧差法，减少背景干扰
- **ROI 优化**：只截取游戏画面中心区域，提升性能
- **配置文件热加载**：运行中实时修改参数
